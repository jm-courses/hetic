Design Adaptatif != Mise en page adaptative !

Design

Statique ?   Fluide ?   Adaptatif ?   Responsive ?

Techniquement, il faut bien faire la distinction

Le site liquidapsive.com propose de tester visuellement ces différents types de design

Un design statique (ou fixe) se réfère à des dimensions figées (par exemple 960px) quelle que soit la surface de l'écran. La grande majorité des sites web était construite sur cette base avant l'arrivée du Responsive Web Design dans les années 2010.

Un site web fluide est un site web dont toutes les largeurs de colonnes sont exprimées en unités variables (pourcentages, em, vw, etc.) et qui s'adapte généralement automatiquement à la taille de fenêtre, jusqu'à une certaine mesure.

Un design Adaptatif est une amélioration du design statique : les unités de largeur sont fixes, mais différentes selon la taille de l'écran, qui est détectée via CSS3 Media Queries.
Un tel design tient uniquement compte des principaux points de rupture (320px, 480px, 768px, 1024px, etc.) et adapte le gabarit en conséquence. Au final, on se retrouve avec autant de gabarits fixes que de points de ruptures.

Un site web Responsive est une amélioration du design fluide associé à des méthodes CSS3 Media Queries permettant de modifier les styles (ré-organisation de la page par exemple) selon certains critères, pour s'adapter complètement à la taille d'écran, quel que soit le point de rupture.

Exemple : le datepicker

Pratique uniquement pour grands écrans et périphériques dotés d'un pointeur

Nous avons longtemps mis en place des datepickers (calendriers) en JavaScript, facilement utilisables sur navigateur de bureau, mais très difficiles à manier pour un appareil tactile. Appuyer sur la bonne date avec un doigt peut s'avérer assez difficile voire impossible si les zones sont très petites. 
<input type="date">

Petits écrans : fallback vers les éléments natifs du périphérique

Gros avantage : le rendu visuel est contrôlé par le navigateur et le système. De cette manière, leur UI (apparence graphique pour l'utilisateur) est automatiquement optimisée au mieux pour le périphérique utilisé.

Media-Queries

Application de feuilles de style en fonction des périphériques

Types de medias


	'screen'     // Écrans
	'handheld'   // Périphériques mobiles ou de petite taille
	'print'      // Impression
	'aural'      // Synthèses vocales
	'braille'    // Plages braille
	'embossed'   // Imprimantes braille
	'projection' // Projecteurs
	'tty'        // Terminal/police à pas fixe
	'tv'         // Téléviseur
	'all'        // Tous les précédents

Dans le HTML


	<link rel="stylesheet" media="screen" href="style.css">
	<link rel="stylesheet" media="print" href="print.css">
	...

Dans le CSS


	@media print {
	html {
	background-color: white;
	}
	body {
	color: black;
	}
	}

Les lecteurs s'identifiants comme des médias d'impression implémenteront ces styles

Des règles plus précises

Les Medias-Queries


	@media screen and (max-width: 650px) {}
	@media (min-width: 700px) and (orientation: landscape) {}
	@media screen and (aspect-ratio: 16/9) {}

Types de règles


	'width'               // dimension en largeur de la zone d'affichage
	'height'              // dimension en hauteur de la zone d'affichage
	'device-width'        // dimension en largeur du périphérique
	'device-height'       // dimension en hauteur du périphérique
	'orientation'         // orientation du périphérique (portait ou landscape)
	'device-aspect-ratio' // ratio du périphérique de sortie (par exemple 16/9)
	'resolution'          // résolution du périphérique (en dpi, dppx, ou dpcm)
	// ...
